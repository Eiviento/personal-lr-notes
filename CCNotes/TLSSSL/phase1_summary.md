# 第一阶段总结：TLS 握手全景图

## 一、TLS 在协议栈中的位置

```
┌──────────────────────────────────────────────────┐
│              应用层 (HTTP / SMTP / FTP ...)       │
├──────────────────────────────────────────────────┤
│           ★  TLS/SSL 安全层  ★                    │
│  ┌─────────────────┬──────────────────────────┐  │
│  │ Handshake       │  Record Protocol         │  │
│  │ Protocol        │  (加密数据的"信封")        │  │
│  │ (协商参数)       │                          │  │
│  └─────────────────┴──────────────────────────┘  │
├──────────────────────────────────────────────────┤
│              传输层 (TCP — 三次握手)              │
├──────────────────────────────────────────────────┤
│              网络层 (IP)                          │
├──────────────────────────────────────────────────┤
│              链路层                               │
└──────────────────────────────────────────────────┘
```

## 二、TLS 1.2 握手全景图 (2-RTT)

```
   Client                                            Server
     │                                                  │
     │  TCP 三次握手 (你已了解)                           │
     │  ├─ SYN →                                        │
     │  │        ← SYN+ACK ─┤                           │
     │  └─ ACK →                                        │
     │                                                  │
     │ ╔═══════════════ 往返 1 开始 ═══════════════════╗ │
     │ ║                                                ║ │
     │ ║ ── [1] ClientHello ──────────────────────────→ ║ │
     │ ║     • TLS 版本 (1.2)                           ║ │
     │ ║     • Client Random (28 bytes)                 ║ │
     │ ║     • 密码套件列表 (所有我会的算法组合)           ║ │
     │ ║     • 扩展 (SNI, 支持曲线, ...)                 ║ │
     │ ║                                                ║ │
     │ ║ ←── [2] ServerHello ────────────────────────── ║ │
     │ ║     • 选定 TLS 版本 (1.2)                      ║ │
     │ ║     • Server Random (28 bytes)                 ║ │
     │ ║     • 选定密码套件                             ║ │
     │ ║        如 TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256║ │
     │ ║                                                ║ │
     │ ║ ←── [3] Certificate ────────────────────────── ║ │
     │ ║     • 服务器证书链 (含公钥)                     ║ │
     │ ║       服务器证书 ← Intermediate CA ← Root CA    ║ │
     │ ║       ↑ 第二阶段重点讲解                        ║ │
     │ ║                                                ║ │
     │ ║ ←── [4] ServerKeyExchange ──────────────────── ║ │
     │ ║     • DH 参数 + 服务器 DH 公钥                  ║ │
     │ ║     • 对参数的签名 (用证书私钥)                  ║ │
     │ ║       ↑ 客户端用证书公钥验证                    ║ │
     │ ║                                                ║ │
     │ ║ ←── [5] ServerHelloDone ────────────────────── ║ │
     │ ║     • (空消息, "我说完了")                      ║ │
     │ ║                                                ║ │
     │ ╚═══════════════ 往返 1 结束 ═══════════════════╝ │
     │                                                  │
     │  [客户端内部处理]                                   │
     │  • 验证证书链                                     │
     │  • 验证 ServerKeyExchange 签名                    │
     │  • 计算 DH 共享密钥 → Pre-Master Secret            │
     │  • PRF(Client Random, Server Random, PMS)         │
     │    → Master Secret → Session Keys                 │
     │                                                  │
     │ ╔═══════════════ 往返 2 开始 ═══════════════════╗ │
     │ ║                                                ║ │
     │ ║ ── [6] ClientKeyExchange ────────────────────→ ║ │
     │ ║     • 客户端 DH 公钥                            ║ │
     │ ║     • (服务器用这个 + 自己的DH私钥 → 相同的PMS)   ║ │
     │ ║                                                ║ │
     │ ║ ── [7] ChangeCipherSpec ──────────────────────→ ║ │
     │ ║     • "从这条以后，我发的全部加密"               ║ │
     │ ║                                                ║ │
     │ ║ ── [8] Finished (加密) ───────────────────────→ ║ │
     │ ║     • 握手所有消息的哈希值 (verify_data)         ║ │
     │ ║     • 用刚协商的 Client Write Key 加密          ║ │
     │ ║     • 作用: 验证整握手过程未被篡改              ║ │
     │ ║                                                ║ │
     │ ╚═══════════════ 往返 2 客户端部分结束 ═══════════╝ │
     │                                                  │
     │  [服务器内部处理]                                   │
     │  • 解密并验证 Finished                             │
     │  • 计算 Session Keys (和客户端算出来的一样)         │
     │                                                  │
     │ ╔═══════════════ 往返 2 服务器回应 ═══════════════╗ │
     │ ║                                                ║ │
     │ ║ ←── [9] ChangeCipherSpec ────────────────────── ║ │
     │ ║     • "收到，我也开始加密"                       ║ │
     │ ║                                                ║ │
     │ ║ ←── [10] Finished (加密) ────────────────────── ║ │
     │ ║     • 服务器端的握手哈希验证                      ║ │
     │ ║     • 用 Server Write Key 加密                  ║ │
     │ ║                                                ║ │
     │ ╚═══════════════ 握手完成！ ══════════════════════╝ │
     │                                                  │
     │ ════════ [加密的 HTTP 请求和响应] ═══════════════ │
     │                                                  │
     │  之后所有数据:                                     │
     │  应用数据 → 分片 → 用 Write Key 加密 → AEAD 认证   │
     │       → 封装成 Record (type=23) → TCP 发送       │
     │                                                  │
```

## 三、TLS 1.3 握手对比 (1-RTT)

```
   Client                                            Server
     │                                                  │
     │ ── [1] ClientHello (增强版) ──────────────────→  │
     │     • 支持的密钥交换曲线                           │
     │     • ★ 每条曲线的 DH 公钥 (Key Share)            │
     │       ↑ 预判服务器会选哪条曲线，提前给               │
     │     • 支持的 AEAD 密码套件                         │
     │                                                  │
     │ ←── [2] ServerHello + 证书 + Finished ────────   │
     │     • 选定曲线 + Server DH 公钥                   │
     │     • 证书链 (加密传输!)                          │
     │     • CertificateVerify (签名)                   │
     │     • Finished                                   │
     │     ↑ 服务器一步到位，密钥交换已完成               │
     │                                                  │
     │ ── [3] Finished ─────────────────────────────→  │
     │                                                  │
     │ ════════ 加密通信 ═══════════════════════════════ │
     │                                                  │
     │ 总耗时: 1-RTT (vs TLS 1.2 的 2-RTT)              │
     │ 如果猜错曲线 → HelloRetryRequest → 变成 2-RTT     │
```

## 四、密钥派生链

```
Client Random (28B) + Server Random (28B)
              +
      Pre-Master Secret
  (DH 协商得出 / RSA 加密传输)
              ↓
         PRF (伪随机函数)
              ↓
      Master Secret (48 bytes)
              ↓
         PRF 派生
              ↓
   ┌──────────────────────────────┐
   │      Session Keys            │
   │                              │
   │  Client Write Key    ← 客户端发数据的对称密钥      │
   │  Server Write Key    ← 服务器发数据的对称密钥      │
   │  Client Write MAC Key ← 客户端数据的 MAC 密钥     │
   │  Server Write MAC Key ← 服务器数据的 MAC 密钥     │
   │  Client Write IV      ← 客户端 CBC 模式的 IV     │
   │  Server Write IV      ← 服务器 CBC 模式的 IV     │
   └──────────────────────────────┘
```

## 五、Record Protocol 封装

```
发送端:                          接收端:
                                  
HTTP 数据 (明文)                  还原的 HTTP 数据
    ↓                                ↑
分片 (≤16KB)                     重组分片
    ↓                                ↑
AEAD 加密+认证 ───────────────→  AEAD 解密+认证验证
    ↓                                ↑
Record 封装                       Record 解析
(type=23, version, length)        (确认 type/version/length)
    ↓                                ↑
交给 TCP ─────────────────────→ 从 TCP 收到
```

## 六、三大目标回顾

| 目标 | TLS 握手中的体现 | TLS 传输中的体现 |
|------|-----------------|-----------------|
| **机密性** | 密钥交换 (DH/RSA) 安全协商会话密钥 | 对称加密 (AES-GCM/ChaCha20) |
| **完整性** | Finished 消息防篡改握手本身 | AEAD 认证标签防篡改每一条 Record |
| **身份认证** | 证书链验证 + 签名验证 | Alert (bad_certificate) |

## 七、10 条握手消息速记

```
往返1:  C→S  ClientHello          ← "客先招呼"
        C←S  ServerHello          ← "服选方案"
        C←S  Certificate          ← "亮证书"
        C←S  ServerKeyExchange    ← "给参数"
        C←S  ServerHelloDone      ← "说完收工"

往返2:  C→S  ClientKeyExchange    ← "客给参数"
        C→S  ChangeCipherSpec     ← "切加密"
        C→S  Finished             ← "验指纹"
        C←S  ChangeCipherSpec     ← "服切加密"
        C←S  Finished             ← "验指纹"
```
