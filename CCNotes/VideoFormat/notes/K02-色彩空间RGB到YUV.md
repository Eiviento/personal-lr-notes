# K02 — 色彩空间：从 RGB 到 YUV

> 学习日期：2026-07-11
> 前置依赖：K01

---

## 1. 为什么视频不用 RGB？

RGB 的三个通道同等重要，无法差异化处理。而人眼对不同信息的敏感度是不同的——这为差异化压缩提供了可能。

---

## 2. 人眼的秘密：亮度 vs 色度

| 感光细胞 | 数量 | 功能 | 特点 |
|----------|------|------|------|
| 视杆细胞（Rods） | ~1.2 亿 | 感知亮度（明暗） | 极度敏感，分布在视网膜周边 |
| 视锥细胞（Cones） | ~600 万 | 感知颜色 | 需较多光，集中在视野中心 |

**核心结论**：人眼对亮度变化极度敏感，对颜色细节相对迟钝。亮度承载了图像的主要结构信息，色度只是"涂色"。

> 这就是视频压缩的生理基础——分离亮度和色度，对色度做更大幅度的压缩。

---

## 3. YUV 色彩空间

| RGB | YUV |
|-----|-----|
| R — 红色分量 | Y — 亮度（Luma），只看 Y = 黑白图像 |
| G — 绿色分量 | Cb（U）— 蓝色色度分量 |
| B — 蓝色分量 | Cr（V）— 红色色度分量 |

**关键洞察**：Y 通道就是一张完美的黑白图像。Cb/Cr 只是给这张黑白图"上色"的数据。

### YUV vs YCbCr

- **YUV**：模拟视频时代的历史术语，日常交流中广泛使用
- **YCbCr**：数字视频中的正式名称（Cb=蓝色色度，Cr=红色色度）
- 日常交流中两者可混用；数字视频领域严格来说叫 YCbCr

---

## 4. RGB ↔ YUV 转换公式（定性理解）

```
Y  = 0.299×R + 0.587×G + 0.114×B     → 亮度 ≈ 加权平均
Cb = 0.564×(B - Y)                     → 蓝色偏离亮度的程度
Cr = 0.713×(R - Y)                     → 红色偏离亮度的程度
```

要点：
- **Y 主要由 G 贡献**（0.587）——人眼对绿光最敏感
- **Cb/Cr 是差分信号**——表示颜色偏离灰色的程度
- 灰像素（R=G=B）时，Cb=0, Cr=0——没有色度信息

---

## 5. 为什么 YUV 是视频编码的起点

```
RGB → 转 YUV → Y 保留全精度 → UV 可以降低精度（人眼不敏感）
                                ↓
                          这就是色度采样（K03）
                          数据量直接减 33%~50%，画质几乎无差异
```

---

## 逻辑链

```
RGB 三通道同等重要 → 人眼对亮度敏感、对色度迟钝 → YUV 分离亮度与色度
→ Y 保留（结构信息）→ UV 可压缩（上色信息）
→ 压缩的第一抓手
```
