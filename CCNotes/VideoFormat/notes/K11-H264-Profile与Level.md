# K11 — H.264 Profile 与 Level

> 学习日期：2026-07-11
> 前置依赖：K08

---

## 1. 问题

为什么同样的 H.264 视频，有些设备能放有些不能？→ Profile 和 Level 不匹配。

---

## 2. 职责划分

| | Profile | Level |
|--|---------|-------|
| 管什么 | 编码用了哪些工具（功能集） | 解码器需要多强性能 |
| 类比 | 工具箱里装了哪些工具 | 力气多大才能拎起这箱子 |
| 关系 | 正交，同一 Profile 可有多个 Level |

---

## 3. Profile 详解

| Profile | 工具集 | 场景 |
|---------|--------|------|
| **Baseline** | I+P 帧、CAVLC、无 B 帧、无 CABAC | 视频会议、低端设备 |
| **Main** | +B 帧、+CABAC | 广播电视（DVB） |
| **High** | +8×8 变换、+自定义量化矩阵 | 蓝光、YouTube、流媒体（**默认选这个**） |
| High 10 | +10-bit 色深 | 专业影视 |
| High 4:2:2/4:4:4 | +4:2:2/4:4:4 采样 | 母带制作 |

**选择原则**：现代场景默认 High，视频会议 Baseline，兼容老设备 Main。

---

## 4. Level 详解

Level 定义硬性性能上限：最大分辨率、帧率、码率、参考帧数。

| Level | 最大分辨率@帧率 | 最大码率 | 场景 |
|-------|---------------|---------|------|
| 3.0 | 720×480@30fps | 10 Mbps | DVD |
| 3.1 | 1280×720@30fps | 14 Mbps | 720p |
| **4.0** | 1920×1080@30fps | 20 Mbps | **1080p 标准** |
| **4.1** | 1920×1080@30fps | 50 Mbps | **蓝光、YouTube** |
| 5.0 | 2560×1920@30fps | 135 Mbps | 2.5K |
| **5.1** | 3840×2160@30fps | 240 Mbps | **4K** |
| 6.1 | 7680×4320@30fps | 480 Mbps | 8K |

---

## 5. 兼容性判断

```
能播放 = Profile 兼容 AND Level 兼容

案例 1：老手机(只支持 Baseline@L3.0) 播 High@L4.1 → 打不开
案例 2：1080p 平板(最高 L4.0) 播 4K(L5.1) → 黑屏/花屏
案例 3：YouTube 1080p 选 High@L4.0 而非 L3.2 → L4.0 码率上限更高
```

---

## 6. 典型组合

| 场景 | Profile@Level |
|------|--------------|
| YouTube 1080p | High@L4.0 |
| 蓝光光盘 | High@L4.1 |
| 4K 流媒体 | High@L5.1 |
| 视频会议 | Baseline@L3.0 |
