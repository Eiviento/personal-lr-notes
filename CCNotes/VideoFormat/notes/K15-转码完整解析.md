# K15 — 转码（Transcoding）完整解析

> 学习日期：2026-07-11
> 前置依赖：K08, K13

---

## 1. 定义

**转码 = 解码 + 重编码。** 把压缩数据展开为原始像素，再按新参数重新压缩。

---

## 2. 完整流程

```
输入 → Demux → 解码器(H.264→YUV) → [可选滤镜] → 编码器(YUV→H.265) → Mux → 输出
```

---

## 3. 为什么必然有损

量化不可逆（K10）。转码 = 第一次量化信息丢失 + 第二次量化再丢 = **代际损失**叠加。

---

## 4. 六大场景

| 场景 | 示例 |
|------|------|
| 编码格式转换 | H.264→H.265 省空间 |
| 分辨率缩放 | 4K→1080p |
| 帧率转换 | 60fps→30fps |
| 码率调整 | 20Mbps→5Mbps |
| 画面处理 | 字幕烧录、水印、裁剪、旋转 |
| 色彩转换 | HDR→SDR、10-bit→8-bit |

---

## 5. FFmpeg 实战

```bash
# 基础转码
ffmpeg -i in.mp4 -c:v libx264 -crf 23 -preset medium -c:a aac -b:a 128k out.mp4

# 转码+缩放
ffmpeg -i in_4k.mp4 -vf scale=1920:1080 -c:v libx264 -crf 23 out_1080p.mp4

# H.264→H.265
ffmpeg -i in.mp4 -c:v libx265 -crf 23 -preset medium out.mp4

# 硬件加速
ffmpeg -i in.mp4 -c:v h264_nvenc -cq 23 out.mp4
```

---

## 6. 时间代价（10 分钟 1080p 视频）

| 方式 | 耗时 |
|------|------|
| 转封装 | ~3 秒 |
| 硬编码 NVENC | ~30 秒 |
| x264 medium | 3-5 分钟 |
| x265 medium | 8-15 分钟 |
| x265 veryslow | 30-60 分钟 |

---

## 7. 决策树

```
只换容器 → 转封装（-c copy，快无损）
改分辨率/帧率/编码/添加画面处理 → 必须转码（慢有损）
```
