## 2. 码率定义

码率 = 每秒多少 bit 用于表示视频。

单位：bps → Kbps（千）→ Mbps（百万）→ Gbps（十亿）

固定分辨率下，码率直接决定画质。

---

## 3. 三种码率控制策略

### CBR（Constant Bitrate，恒定码率）
- 每秒分配固定数量的比特
- 优点：文件大小精确可预测、适合固定带宽传输
- 缺点：静止画面浪费码率，复杂画面码率不够
- 场景：直播推流（RTMP）、视频会议

### VBR（Variable Bitrate，可变码率）
- 复杂画面多给码率，简单画面少给
- 优点：同平均码率下画质优于 CBR
- 缺点：文件大小不可精确预测，瞬时码率可能很高
- 场景：视频网站（YouTube/Netflix）

### CRF（Constant Rate Factor，恒定质量）
- x264/x265 特有，你控制质量，编码器自动决定码率
- CRF 范围 0-51：18≈视觉无损，23≈默认，28≈可接受低质量
- 不同内容同 CRF 产出不同码率（动画 vs 动作片差异大）
- 场景：个人视频归档、压制发布

---

## 4. 三种策略对比

| | CBR | VBR | CRF |
|------|-----|-----|-----|
| 码率可预测性 | ★★★ | ★★ | ★ |
| 画质稳定性 | ★☆ | ★★★ | ★★★ |
| 文件大小可预测 | ★★★ | ★☆ | ★ |
| 典型场景 | 直播 | 视频网站 | 归档 |

## 5. 压缩进展

```
未压缩 RGB          → 1.5 Gbps
转 YUV 4:2:0        → ~746 Mbps  (×2，色彩空间+采样)
H.264 编码后         → ~5-10 Mbps  (×75-150，帧内+帧间+熵编码)
```
