# K07 — MJpeg（Motion JPEG）深度解析

> 学习日期：2026-07-11
> 前置依赖：K01, K04

---

## 1. MJpeg 是什么

**MJpeg = 把每一帧存成独立 JPEG 图片，连续播放。** 不是复杂视频编码标准，没有 I/P/B 帧，就是 JPEG 图片串。

---

## 2. JPEG 压缩流程（回顾）

```
原始图像(RGB) → 色彩空间转换(RGB→YCbCr) → 色度采样(4:2:0/4:2:2)
→ 分块(8×8) → DCT 变换(空间域→频域) → 量化(丢高频，有损关键)
→ 熵编码(霍夫曼编码，数学无损)
```

量化是核心：平坦区域低频多→保留好；纹理区域高频多→大量丢弃。蓝天图片比草地图片小得多。

---

## 3. 帧内编码 vs 帧间编码

| | MJpeg（纯帧内） | H.264（帧内+帧间） |
|--|---------------|-------------------|
| 帧间关系 | 每帧独立 | I/P/B 帧依赖 |
| 相同内容 3 帧 | 3 × 100KB = 300KB | 100 + 20 + 20 = 140KB |
| 随机访问 | 任意帧直达 | 需回溯最近 I 帧 |

---

## 4. 五大优势

| 优势 | 说明 |
|------|------|
| 帧独立 | 任意帧随机访问，不需解码前序帧 |
| 丢帧不扩散 | 损坏一帧不影响后续帧 |
| 编辑友好 | 精确到帧切割 |
| 低延迟 | 不需等未来帧、不需运动估计搜索 |
| 实现简单 | 只需 JPEG 芯片，内存占用小 |

---

## 5. 两大劣势

- **码率 ≈ H.264 的 3~5 倍**：不利用帧间冗余，每一帧"重复发明轮子"
- **没有统一视频标准**：JPEG 图片本身标准化，但"怎么串成视频"各厂商不同，可能不兼容

---

## 6. 典型应用场景

| 场景 | 为什么用 MJpeg |
|------|---------------|
| 监控实时预览 | 低延迟 + 丢帧不扩散 + 任意时刻能截图 |
| 无人机 FPV 图传 | 极低延迟 + 信号中断后秒恢复 |
| 医学影像（内窥镜/超声） | 每帧绝对准确，不能有伪影 |
| 编辑中间格式 | 精确到帧剪辑、快速 scrubbing |
| 早期数码相机 | 硬件简单便宜 |

常见模式：**实时预览用 MJpeg，长期存储用 H.264**。

---

## 7. 封装形式

- 文件：`.mjpeg` / `.avi`(MJpeg编码) / `.mov`(MJpeg编码)
- 网络流：`video/x-motion-jpeg`（HTTP MJPEG 流，每收到一个 JPEG 刷新一次画面）
