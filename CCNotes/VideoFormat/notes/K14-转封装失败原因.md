# K14 — 为什么转封装有时会失败

> 学习日期：2026-07-11
> 前置依赖：K13

---

## 四大失败原因

### ① 容器不支持编码格式（最常见）

| 编码 | MP4 | MKV | TS | WebM |
|------|-----|-----|----|------|
| H.264 | ✅ | ✅ | ✅ | ❌ |
| H.265 | ✅ | ✅ | ✅ | ❌ |
| VP8/VP9 | ❌ | ✅ | ❌ | ✅ |
| AV1 | △ | ✅ | ❌ | ✅ |
| AAC | ✅ | ✅ | ✅ | ❌ |
| Opus | ❌ | ✅ | ❌ | ✅ |
| Vorbis | ❌ | ✅ | ❌ | ✅ |
| FLAC | ❌ | ✅ | ❌ | ❌ |

解决：换容器（如 MKV 装一切）或对不兼容流转码。

### ② 裸流打包格式不匹配

同一种编码有不同打包方式：
- **Annex B**：起始码 0x00000001 分隔 NALU（TS、裸流文件）
- **AVCC**：长度前缀分隔 NALU（MP4/MKV/MOV）

转换用 bitstream filter：
```bash
ffmpeg -i input.mp4 -c:v copy -bsf:v h264_mp4toannexb output.h264  # AVCC→Annex B
```

### ③ 时间基不匹配（罕见）

不同容器默认时间基不同（MP4=1/90000, MKV=1/1000, TS=1/90000），转换时舍入误差可能导致音视频不同步。FFmpeg 通常自动处理。

### ④ MP4 moov atom 位置

moov 在文件末尾 → 网络播放需下载完整文件。解决：
```bash
ffmpeg -i input.mp4 -c copy -movflags +faststart output.mp4
```

## 排查工具

```bash
ffprobe -v error -show_entries stream=codec_name,codec_type input.mkv  # 看编码
ffmpeg -i input.mkv -c copy -f mp4 /dev/null  # 试运行，看是否报错
```
