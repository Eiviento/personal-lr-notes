# 8.1 OPPO Enco Air2 练手 App — 配置说明

## 项目创建

在 Android Studio 中创建项目：

```
Template: Empty Views Activity
Package:  com.example.blelab
Language: Java
Min SDK:  API 24 (Android 7.0)
```

## 文件替换

将 `MainActivity.java` 复制到 `app/src/main/java/com/example/blelab/` 覆盖自动生成的文件。

**不需要 `activity_main.xml`** — UI 全部在代码中动态构建。

## AndroidManifest.xml

```xml
<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android">

    <!-- Android 12+ 专用 BLE 权限 -->
    <uses-permission android:name="android.permission.BLUETOOTH_SCAN" />
    <uses-permission android:name="android.permission.BLUETOOTH_CONNECT" />

    <!-- 基础蓝牙权限（Android 11-） -->
    <uses-permission android:name="android.permission.BLUETOOTH" />
    <uses-permission android:name="android.permission.BLUETOOTH_ADMIN" />

    <!-- Android 6-11: BLE 扫描强制要求定位权限 -->
    <!-- 这就是"开 GPS 才能扫 BLE"的协议层根源 -->
    <uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
    <uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />

    <!-- 声明 BLE 硬件特性 -->
    <uses-feature android:name="android.hardware.bluetooth_le" android:required="true" />

    <application
        android:allowBackup="true"
        android:label="BLE练手"
        android:theme="@style/Theme.AppCompat.Light">

        <activity
            android:name=".MainActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>

    </application>
</manifest>
```

## 依赖

不需要额外依赖。纯 Android SDK `android.bluetooth` + `android.bluetooth.le` 包，API 24 以上原生支持。

## 操作流程

```
1. 打开 App → 确保手机蓝牙已开 → 自动请求权限
2. 戴上耳机让 OPPO 处于广播状态
3. 点 "🔍 扫描连接" → 自动匹配并连接
4. 连接成功后按钮解锁:
   - 📋 读设备信息 → 读制造商/固件/型号
   - 📖 读1235 → 读厂商特征值
   - ✏️ 写FF02 → 写 ATT WRITE_CMD 测试
   - 🔔 订阅1236 → 写 CCCD 开启通知
5. 观察 Logcat 过滤 "BLE_LAB" 查看协议日志
6. 点断开 → 重连 → 验证 CCCD 丢失现象
```

## Logcat 过滤

```
adb logcat -s BLE_LAB
```

或在 Android Studio Logcat 中过滤 tag:`BLE_LAB`。

## 关键验证实验

| 实验 | 操作 | 预期结果 |
|------|------|---------|
| MTU 顺序 | 正常连接 | 日志显示 MTU 协商在服务发现之前 |
| CCCD 两步 | 点订阅 | 日志先 setNotification 再 writeDescriptor |
| 通知到达 | 订阅后触发耳机事件 | `onCharacteristicChanged` 回调，日志显示 `🔔 通知!` |
| CCCD 丢失 | 断开→重连→**不点订阅** | 即使设备发通知，`onCharacteristicChanged` 不触发 |
| CCCD 恢复 | 重连后点订阅 | 通知恢复正常 |

## 注意事项

1. **Android 10 以下**：必须打开 GPS 才能扫描 BLE（定位权限要求）
2. **OPPO 耳机**：可能需戴上耳朵或开盖才能被扫描到
3. **写入 FF02/FF03**：厂商私有特征值，写入可能触发实际行为（如 ANC 切换），建议只写 `0x01` 测试
4. **读取失败**：某些特征值需要配对/加密才能读，`status=5` 表示 `Insufficient Authentication`
5. **连接失败**：如果耳机已被手机系统占用，BleManager 连接会被拒绝。先断开系统蓝牙配对再试
