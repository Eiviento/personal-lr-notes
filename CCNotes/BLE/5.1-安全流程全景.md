# 5.1 安全流程全景

## 一、BLE 安全四步曲

```
配对(Pairing) → 加密(Encryption) → 密钥分发(Key Distribution) → 绑定(Bonding)
                                                                    │
                                    下次连接直接用 LTK 加密 → 跳过配对！
```

配对只做一次。绑定后重连自动加密，对用户无感。

## 二、每步详解

### 配对
Phase 1: 交换 IO Capabilities → 协商配对方式
Phase 2: 生成密钥（Legacy 先生成 STK 再生成 LTK，SC 直接生成 LTK）

### 加密
链路层用 LTK 加密链路，对上层 ATT/GATT 透明。

### 密钥分发
| 密钥 | 用途 |
|------|------|
| LTK | 加密链路 |
| EDIV+Rand | LTK 索引 |
| IRK | 解析私有地址 |
| CSRK | 数据签名 |

### 绑定
密钥 + 对端身份信息存入 Flash（非易失存储）。通常存 5~10 个设备。

## 三、安全模式与等级

### Mode 1（需要加密）
| Level | 要求 |
|-------|------|
| 1 | 无安全 |
| 2 | 加密但未认证 (Just Works) |
| 3 | 加密+认证 (Passkey/Numeric/OOB) |
| 4 | LE Secure Connections 加密+认证 |

### Mode 2（数据签名）
Level 1: 未认证签名 / Level 2: 已认证签名

### 流量计典型设置
- 设备名/序列号 → Level 1（公开）
- 流速/累计流量 → Level 2~3（需加密）
- 校准参数/固件升级 → Level 4（需 SC+认证）

## 四、Android App 视角

**已绑定设备**：连接→协议栈自动用 LTK 加密→正常读写（无感）

**首次连接**：连接→读需加密的 Characteristic→Android 自动触发配对→用户确认→加密→绑定→正常读写

App 通常不需要手动触发配对——Android 协议栈在读需要加密的数据时自动启动。
