# 3.4 广播参数

## 一、时间参数

### Advertising Interval
两次广播事件间隔：20ms ~ 10240ms，单位 0.625ms。
- 短间隔 → 发现快但功耗高
- 长间隔 → 省电但发现慢

### AdvDelay
每次间隔加 0~10ms 伪随机延迟。**目的**：防止两个同间隔设备永久碰撞——随机延迟让它们在几个周期后自然错开。

### Scan Window / Scan Interval（Central 侧）
- Scan Window：每次扫描持续时长（在此窗口内在 3 个信道间轮转）
- Scan Interval：两次扫描间隔
- 两设备相遇条件：广播包落在扫描窗口内且在同一信道

## 二、广播数据：AD Structure 格式

AdvData (31字节) 用 LTV 格式（Length-Type-Value）组织：

```
┌──────────┬──────────┬─────────────────────┐
│ Length   │ AD Type  │ AD Data             │
│ (1字节)  │ (1字节)  │ (Length-1 字节)     │
└──────────┴──────────┴─────────────────────┘

多个 AD Structure 依次拼接：
02 01 06        ← Flags: 仅BLE + 普通可发现
03 03 0F 18     ← 16-bit Service UUID: 0x180F (电池服务)
09 09 46 4C...  ← Complete Local Name: "FLOWMETER"
```

### 常用 AD Type

| Type | 名称 | 说明 |
|------|------|------|
| 0x01 | Flags | LE 模式标志位 |
| 0x02/0x03 | 16-bit Service UUIDs | 部分/完整 |
| 0x06/0x07 | 128-bit Service UUIDs | 部分/完整 |
| 0x08/0x09 | Local Name | 短名/完整名 |
| 0x0A | TX Power Level | 用于距离估算 |
| 0xFF | Manufacturer Specific | 厂商自定义 |

## 三、ScanRspData

额外 31 字节，仅在主动扫描时通过 SCAN_RSP 返回。AdvData + ScanRspData = 最多 62 字节。

## 四、BLE 5.0 广告扩展

打破 31 字节限制：每包最大 251 字节，支持链式广播，总数据可达 1650 字节。需要双方支持 BLE 5.0。

## 五、问答记录

### Q1: SCAN_REQ/SCAN_RSP 过程中设备状态如何变化？从什么广播类型转到什么扫描类型？

**双方状态完全不变。** Peripheral 始终在 Advertising，Central 始终在 Scanning。SCAN_REQ/SCAN_RSP 只是一次广播事件内部的子交互。

ADV_IND 本身同时支持被连接和被扫描，不需要切换类型。Central 发不发 SCAN_REQ 是可选的——发就是主动扫描，不发就是被动扫描。

**一次信道上完整时序**：Peripheral 发 ADV_IND → 短暂打开接收窗口（百微秒级）→ 收到 SCAN_REQ 则回 SCAN_RSP，收到 CONNECT_IND 则进入 Connection，啥也没收到则休眠等下次广播。

### Q2: 如果 AdvData 用满了，还需要做设备隔离，有什么方案？

四个方案，从轻到重：

| 方案 | 广播数据 | 安全性 | 防恶意连接 |
|------|---------|--------|-----------|
| ScanRspData UI 过滤 | 不占 AdvData | 无 | ❌ |
| GATT 认证门禁 | 0 字节 | 中 | ❌ |
| 配对+白名单 | 0 字节 | 高 | ✅ 链路层拒绝 |
| 物理按钮+定向广播 | 0 字节 | 高 | ✅ |

**推荐混合方案**：GATT 认证 + 配对白名单。日常用白名单自动加密连接（用户无感），新设备通过 GATT 认证密钥或标准配对完成初始配置。
