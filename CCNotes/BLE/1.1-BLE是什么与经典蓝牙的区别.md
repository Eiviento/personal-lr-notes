# 1.1 BLE 是什么 & 与经典蓝牙的区别

## 一、蓝牙技术的两个分支

蓝牙技术发展到 4.0 版本时，分裂成了两个"性格迥异"的分支：

```
                    蓝牙 (Bluetooth)
                         │
         ┌───────────────┴───────────────┐
         ▼                               ▼
   经典蓝牙 (BR/EDR)              低功耗蓝牙 (BLE)
   Basic Rate /                   Bluetooth Low Energy
   Enhanced Data Rate
```

**它们共享同一个名字"蓝牙"、同一个 2.4GHz 频段、同一套品牌，但实际上是完全不同的两套协议。**

---

## 二、核心区别对比

| 维度 | 经典蓝牙 (BR/EDR) | 低功耗蓝牙 (BLE) |
|------|-------------------|------------------|
| **设计目标** | 持续传输大量数据 | 间歇传输少量数据 |
| **功耗** | 高（几十 mA） | 极低（几 μA ~ 十几 mA） |
| **数据速率** | 1~3 Mbps | 125 Kbps ~ 2 Mbps |
| **连接建立时间** | ~100ms | ~3ms |
| **最大数据包** | 较大（可达 1021 字节/包） | 较小（27~251 字节/包） |
| **信道数** | 79 个 | 40 个（3 广播 + 37 数据） |
| **网络拓扑** | 点对点 + 微微网（Piconet） | 点对点 + 广播 + Mesh |
| **典型场景** | 音频（耳机、音箱）、文件传输、车载电话 | 传感器、穿戴设备、定位信标、智能家居、仪器仪表 |
| **规范版本** | 蓝牙 1.0 ~ 3.0 的主力 | 从蓝牙 4.0 引入（2010年） |
| **芯片成本** | 较高 | 极低（不到 $1） |

---

## 三、关键的理解：为什么 BLE 功耗低？

这是 BLE 之所以叫"Low Energy"的根本原因，理解这几点对你后续学习很重要：

### 3.1 "睡"得多、"醒"得少（Duty Cycle）

```
经典蓝牙：
  ████████████████████████████████████████  ← 持续活跃
  时间 ──────────────────────────────────→

BLE：
  ██___ ██___ ██___ ██___ ██___ ██___  ← 周期性短暂活跃
  时间 ──────────────────────────────────→
```

BLE 设备 99% 以上的时间处于休眠状态，只在约定好的时间点"醒来"交换数据。

### 3.2 连接建立极快

经典蓝牙建立连接需要先 Inquiry（设备发现）→ Page（寻呼）→ 连接，整个过程约 100ms。BLE 扫描到广播→直接发送连接请求，只需约 3ms。**快速建立 → 快速交换 → 快速休眠**，减少了射频开启时间。

### 3.3 数据包精简

经典蓝牙数据包 header 开销大，BLE 的 packet 设计极其精简：
- 广播包只需几十字节
- 数据包头部仅 2 字节

### 3.4 自适应跳频

只在"干净"的信道上通信（37 个数据信道中，避开干扰严重的信道），减少重传，将射频工作时间降到最低。

## 四、历史版本演进（理解版本号的含义）

| 版本 | 年份 | 对 BLE 的关键变化 |
|------|------|------------------|
| **蓝牙 4.0** | 2010 | BLE 首次引入，仅支持 1M PHY |
| **蓝牙 4.1** | 2013 | 改进共存性（与 LTE 协调信道）、支持双模设备同时扮演 Central 和 Peripheral |
| **蓝牙 4.2** | 2014 | **LE Secure Connections**（安全大幅提升）、LE Data Length Extension（单包 251 字节）、链路层隐私（Private Address） |
| **蓝牙 5.0** | 2016 | **2M PHY**（速度翻倍）、**LE Coded PHY**（远距离，4倍）、广播扩展（Advertising Extensions，不再限于 31 字节）、Slot Availability Mask |
| **蓝牙 5.1** | 2019 | **AoA/AoD 测向**（到达角/出发角，室内定位精度亚米级） |
| **蓝牙 5.2** | 2020 | **LE Audio**（LC3 编解码）、EATT（增强 ATT）、LE Isochronous Channels |
| **蓝牙 5.3** | 2021 | 连接子额定（Channel Classification 改进）、周期性广播增强 |
| **蓝牙 5.4** | 2023 | Periodic Advertising with Responses（PAwR），为电子货架标签优化 |

> **对你来说**：公司设备大概率至少支持蓝牙 4.2 或 5.0。做仪器仪表类产品，5.0 的 2M PHY（更快读数）和 Advertising Extensions（更多广播数据）比较实用。

---

## 五、双模 vs 单模

- **双模设备**（Dual Mode / BR/EDR/LE）：同时支持经典蓝牙和 BLE。比如手机，既能连蓝牙耳机（经典），也能连智能手表（BLE）。
- **单模设备**（Single Mode / BLE-only）：只支持 BLE。市面上绝大多数 IoT 设备、传感器都是这种。

你们的流量计大概率是 **BLE 单模**设备。

---

## 六、一句话总结

> **BLE 不是"省电版经典蓝牙"，而是一套从零设计的、为低功耗场景优化的全新协议。它牺牲了持续高带宽，换来了极低功耗、极快连接和极低芯片成本。**
