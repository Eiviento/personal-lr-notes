# 6.2 ATT PDU 类型全解

## 一、分类总览（12 种核心 PDU）

**读操作（5种）**：Read Request/Response、Read Blob（长数据分段读）、Read By Type（按UUID类型搜索Characteristic）、Read By Group Type（按组搜索Service）、Find Information（发现所有属性信息/Descriptor）

**写操作（4种）**：Write Request（带确认）、Write Command（无确认，Opcode bit6=1）、Signed Write Command（CSRK签名，需加密链路）、Prepare+Execute Write（事务性长写）

**通知/指示（2种）**：Notification（无确认，快）、Indication+Confirmation（需确认，可靠）

**错误响应（1种）**：Error Response

## 二、何时用什么

| | Notification | Indication |
|---|------------|-----------|
| 可靠性 | 不可靠 | 可靠（等CFM） |
| 速度 | 快 | 慢 |
| 流量计场景 | 流速/累计流量（丢了下一包马上到） | 报警事件（不能丢） |

常见错误码：Invalid Handle(0x01) / Read/Write Not Permitted(0x02/0x03) / Insufficient Authentication(0x05) / Insufficient Encryption(0x08)

## 三、Android API 对应

| 场景 | PDU | Android API |
|------|-----|------------|
| 读取当前值 | Read Request | readCharacteristic() |
| 读取长数据 | Read Blob | readCharacteristic() (自动) |
| 写配置(需确认) | Write Request | writeCharacteristic() |
| 快速批量写 | Write Command | writeCharacteristic()+WRITE_TYPE_NO_RESPONSE |
| 设备主动推送 | Notification/Indication | setCharacteristicNotification()+写CCCD |
| 原子性长写 | Prepare+Execute | writeCharacteristic() (自动) |
