# 7.2 广播与扫描的 Android 实现

## 一、扫描流程

`startScan(callback)` → 底层 Scanning → LE Advertising Report → `onScanResult(result)`

## 二、核心代码

```java
ScanSettings settings = new ScanSettings.Builder()
    .setScanMode(ScanSettings.SCAN_MODE_LOW_LATENCY)
    .setReportDelay(0)
    .build();

List<ScanFilter> filters = Arrays.asList(
    new ScanFilter.Builder()
        .setServiceUuid(ParcelUuid.fromString(UUID_STRING))
        .build()
);

scanner.startScan(filters, settings, scanCallback);
```

ScanCallback: onScanResult / onBatchScanResults / onScanFailed

## 三、扫描模式

| 模式 | 类型 | Duty Cycle |
|------|------|-----------|
| LOW_POWER | 被动扫描 | ~10% |
| BALANCED | 主动扫描 | ~40% |
| LOW_LATENCY | 主动扫描高占空比 | ~100% |

## 四、ScanFilter — 硬件层过滤

硬件/协议栈层过滤，比 Java 层省电。支持：按 Service UUID / 设备地址 / 设备名 / 厂商数据过滤。

## 五、ScanRecord 解析

`getDeviceName()` / `getServiceUuids()` / `getManufacturerSpecificData()` / `getTxPowerLevel()` / `getBytes()`

## 六、RSSI 距离估算

不要精确计算距离，做离散分级：>-50 很近 / -50~-70 一般 / -70~-90 较远 / <-90 远

## 七、最佳实践

- 设超时（如10s），不要一直扫描
- 用 ScanFilter 硬件过滤
- 发现目标立刻停扫
- Android 10+ 后台扫描受限（30分钟/30秒）
- 回调里做轻量操作
