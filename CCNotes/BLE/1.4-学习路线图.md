## 三、学习重点分布（时间精力分配建议）

| 阶段 | 重要性 | 时间占比 | 原因 |
|------|--------|---------|------|
| 一：全景概览 | ⭐⭐ | 5% | 建立地图即可，不深究 |
| 二：PHY | ⭐⭐ | 10% | 了解物理约束，知道"为什么这样设计" |
| **三：Link Layer** | ⭐⭐⭐⭐⭐ | **25%** | **BLE 的核心机制都在这里** |
| 四：HCI+L2CAP | ⭐⭐ | 10% | 承上启下的管道 |
| **五：SM** | ⭐⭐⭐ | **15%** | 出问题很难排查，必须理解 |
| **六：ATT/GATT** | ⭐⭐⭐⭐⭐ | **25%** | **你写代码直接打交道的层** |
| 七：GAP+Android | ⭐⭐⭐ | 10% | API 实战，把知识转化为代码 |

```

```

## 五、每层与 Android API 的对应关系

这张表告诉你：**你学的这个知识点，对应哪行 Android 代码？**

| 协议层 | 你学的内容 | Android API 对应 |
|--------|-----------|-----------------|
| Link Layer | 广播/扫描/连接 | `BluetoothLeScanner.startScan()`<br>`BluetoothDevice.connectGatt()` |
| GAP | Central/Peripheral 角色 | 你的手机自动是 Central，API 不暴露角色切换 |
| GATT | Service/Characteristic 发现 | `BluetoothGatt.discoverServices()`<br>`BluetoothGattService.getCharacteristics()` |
| ATT | Read/Write/Notify | `BluetoothGatt.readCharacteristic()`<br>`BluetoothGatt.writeCharacteristic()`<br>`BluetoothGatt.setCharacteristicNotification()` |
| SM | 配对/绑定 | `BluetoothDevice.createBond()`<br>`BluetoothGattCallback.onConnectionStateChange()` |
| L2CAP | 分片/MTU | `BluetoothGatt.requestMtu()` |

> **你现在可能看不懂这些 API，没关系，学到阶段七时会全部串起来。**

## 六、推荐学习的工具和资料

| 资源 | 用途 | 阶段 |
|------|------|------|
| **Bluetooth Core Spec 5.x** | 协议圣经，选读关键章节 | 全阶段 |

| **Bluetooth SIG 官网** | 16-bit UUID 分配表、标准 Service/Characteristic 定义 | 阶段六 |
