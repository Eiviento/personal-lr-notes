# 方法论：概念深度学习提示词模板

> 适用场景：需要从零开始系统学习某个技术领域，逐层拆解、深度理解。
> 提炼自：BLE 协议学习过程（2026-07-11）

---

## 第一阶段：建立学习框架（只需做一次）

用以下提示词启动一个新的学习主题：

```
我是一名 [你的角色/背景]，最近需要 [具体任务/目标]。
但我对 [目标领域] 不懂/了解不深。

请帮我：
1. 先了解我的背景（平台、现有知识水平、学习深度诉求），
   一次只问一个问题，等我回答后再问下一个。
2. 了解清楚后，给我 2-3 种学习路径方案（含优缺点和推荐）。
3. 确定方案后，把这个领域拆解成 [N] 个阶段，
   每个阶段列出具体的知识点任务，让我确认。
4. 确认后把完整计划写入文件，然后创建任务跟踪，
   从第一阶段开始带我学习。
```

> 关键原则：**一次一个问题，逐步收敛**，不要一次性抛出所有问题。

## 第二阶段：逐个知识点深度学习（每个知识点重复此流程）

### 标准提示词（每次开始新知识点时用）

```
请讲解 [知识点名称]。

要求：
1. 先讲"为什么需要这个东西"（动机/背景）
2. 再讲"它是什么"（核心概念 + 结构）
3. 然后用图表/对比表/ASCII 图辅助理解
4. 最后讲"对我开发有什么用"（实际意义）

内容先在对话中发给我，等我确认没问题后再保存为笔记文件。
```

### 进阶提示词（需要更深入时）

```
请详细对比 [概念A] 和 [概念B] 的区别，
用表格列出每个维度的差异，
并说明各自适用的场景。
```

### 追问提示词（有疑问时）

```
我对 [具体点] 不太理解。
请用 [类比/图解/具体例子] 的方式帮我建立直觉。
```

## 第三阶段：笔记记录规范

每个知识点笔记应包含以下结构（可直接作为 AI 输出模板）：

```markdown
# [编号] [标题]

## 一、[背景/动机]  — 为什么要学这个

## 二、[核心概念]  — 它是什么

## 三、[详细拆解]  — 怎么工作的（可多子节）

## 四、[对比分析]  — 与其他相关概念的区别（如有）

## 五、[实际意义]  — 对我开发有什么影响

## 六、[问答记录]  — 学习过程中的提问和回答（如有）

---

*记录时间：YYYY-MM-DD*
*关联：[[相关笔记文件名]]*
```

## 今天对话中验证有效的模式总结

### 有效的做法

| 做法 | 效果 |
|------|------|
| **一次一个问题确定背景** | 用户不会 overwhelmed，每个回答都精准 |
| **自底向上分层** | 每层建立在上一层基础上，不跳跃 |
| **先给方案选项再确认** | 用户有掌控感，方案量身定制 |
| **内容先在对话中呈现** | 用户可以即时提问、修正，笔记是"定稿" |
| **每节末尾设问** | 明确给出"有问题吗？没问题继续"的信号 |
| **问答融入笔记** | 把学习过程中的讨论也保存下来，回头看有上下文 |
| **ASCII 图/表格辅助** | 可视化比纯文字好理解 |
| **每层最后讲"对你有什么用"** | 把理论和开发实践串起来 |

### 需要避免的做法

| 做法 | 问题 |
|------|------|
| 一次性抛出 5 个问题 | 用户不知道从哪个回答 |
| 直接写文件不让用户看 | 用户没法即时纠正理解偏差 |
| 跳过"为什么需要"直接讲细节 | 缺乏动机，知识碎片化 |
| 只讲理论不提实际影响 | 应用开发者会觉得"学了没用" |
| 一次性给几十页内容 | 信息过载，消化不了 |

## 完整对话流程模板

```
第一轮：建立框架
─────────────────
用户: [初始提示词 — 描述背景和目标]
AI:   了解背景（一次一问）
AI:   给出 2-3 种学习路径 → 用户选择
AI:   展示完整知识拆解 → 用户确认
AI:   写入计划文件 + 创建任务

后续每一轮：学习一个知识点
────────────────────────────
AI:   在对话中呈现知识点内容（先不写文件）
      末尾问"有疑问吗？没问题我保存继续"
      
用户: "继续"          → AI 保存笔记 → 进入下一个知识点
用户: "有个问题..."    → AI 回答 → 追问收敛 → 保存（含问答）→ 继续
用户: "今天到这"      → AI 保存当日进度 → 更新计划 → 总结

阶段结束：
─────────
AI:   总结本阶段完成情况 + 预告下一阶段内容
```

## 通用提示词汇总（可直接复制使用）

### 启动新学习主题
```
我是 [角色]，需要 [目标]，但对 [领域] 不了解。
请先了解我的背景再帮我制定学习计划。一次只问一个问题。
```

### 讲解单个知识点
```
请讲解 [知识点]：
- 为什么要学
- 核心概念
- 详细机制
- 与其他概念的区别
- 实际开发意义
先发内容给我看，确认后保存。
```

### 深入对比
```
请对比 [A] 和 [B]，用表格列出每个维度的差异和适用场景。
```

### 提问追问
```
[概念] 中 [具体点] 不太理解，请用类比或图解帮我建立直觉。
```

### 暂停/恢复
```
今天先到这。    ← 暂停
继续。          ← 恢复下一个知识点
```
