# 4.1 HCI — 主机控制器接口

## 一、HCI 位置

Controller (LL+PHY) 和 Host (L2CAP+SM+ATT+GATT) 之间的分界线。Android 手机中走 UART 连接主 CPU 和蓝牙芯片。

## 二、HCI 五类包

| 类型 | 方向 | 用途 | BLE 是否使用 |
|------|------|------|------------|
| Command | Host→Controller | 下发指令 | ✅ |
| Event | Controller→Host | 上报事件 | ✅ |
| ACL Data | 双向 | 承载异步数据(ATT/GATT/SM) | ✅ |
| SCO Data | 双向 | 音频数据 | ❌ (经典蓝牙) |
| ISO Data | 双向 | LE Audio 流数据 | ❌ (5.2+, 流量计不用) |

## 三、包格式

### HCI Command
OpCode(2B) + Parameter Length(1B) + Parameters(变长)
- OGF(6bit) 分组 + OCF(10bit) 组内命令

常用 LE 命令：LE Set Scan Enable / LE Create Connection / LE Set Advertising Enable / LE Disconnect

### HCI Event
Event Code(1B) + Length(1B) + Parameters(变长)

重要 Event：LE Advertising Report（扫到设备）、LE Connection Complete（连接成功）、Disconnection Complete（断开）

### HCI ACL Data
Handle(12bit) + PB Flag(2bit) + BC Flag(2bit) + Data(L2CAP包)

Handle 标识属于哪个连接。

## 四、HCI 传输层

UART(Android手机, 最常见) / USB(PC蓝牙Dongle) / SDIO / SPI

## 五、Android HCI Snoop Log

开发者选项→"启用蓝牙 HCI 信息收集日志"→生成 `btsnoop_hci.log`→Wireshark 打开。所有 HCI 包的时间和流向一览无余，最强的 BLE 调试工具。

## 六、Android API ↔ HCI 对应

| Android API | HCI |
|------------|-----|
| startScan() | LE Set Scan Enable(Enable=1) |
| stopScan() | LE Set Scan Enable(Enable=0) |
| connectGatt() | LE Create Connection → Event: LE Connection Complete |
| disconnect() | LE Disconnect → Event: Disconnection Complete |
| onScanResult() | Event: LE Advertising Report |
