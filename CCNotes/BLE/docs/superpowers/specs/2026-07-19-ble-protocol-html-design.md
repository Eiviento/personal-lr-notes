# Spec: BLE 协议交互报文 HTML 页面

## 概述

基于 `附录-BLE协议交互报文完整手册.md`（1254 行，10 章 + 附录）制作一个**单文件交互式 HTML 页面**，供学校教学使用。

- **目标用户**：学习 BLE 协议的学生
- **使用场景**：个人屏幕阅读 + 教室投影讲解
- **约束**：单文件，无外部依赖（无 CDN、无 node_modules），直接双击打开

## 核心组件规格

### 组件一：位布局可视化

**目标**：直观展示 PDU Header 中每一位的含义和位置。

**实现**：上方色块条 + 下方表格，联动高亮。

**色块条**：
- 总宽度固定，按各字段 bit 数量比例分配宽度
- 冷色系配色：蓝(数据长度)、青(地址相关)、绿(控制位)、灰(RFU)、橙(PDU 类型)
- 每个区块标注：字段名、bit 数、bit 范围（如 `[15:8]`）

**详细表格**：
- 每行一个字段，左侧带有与色块条同色的小色标
- 列：字段名 | 位宽 | 位范围 | 说明（含取值枚举）

**交互**：
- 鼠标悬停色块条区块 → 对应表格行高亮 + 色块轻微放大
- 鼠标悬停表格行 → 对应色块区块高亮

**涉及手册位置**：广告 PDU Header (16-bit)、数据 PDU Header (16-bit)、ATT Opcode (8-bit)、SMP Code (8-bit)、空中帧结构

### 组件二：27 步报文时序图

**目标**：完整展示从广播到断开的所有报文交互，支持逐步查看细节。

**实现**：静态完整时序图 + 点击步骤展开详情 + 可选自动播放。

**布局**：
```
Peripheral (设备B)                     Central (设备A)
     │                                       │
═════╪═══ 广播阶段 (37/38/39) ══════════════╪════   ← 阶段用不同浅色背景区分
     │ ADV_IND ────────────────────────────→│         ← 可点击，悬停显示摘要
     │ ...                                  │
═════╪═══ SMP 配对 ═════════════════════════╪════
     │ ...                                  │
```

**阶段背景色**：广播(浅蓝)、连接(浅绿)、SMP(浅紫)、加密(浅红)、ATT(浅橙)、断开(浅灰)

**交互**：
- 点击某条报文 → 时序图下方展开该报文的完整结构（位布局色块条 + 字段表）
- 每条报文右侧标注层（LL/SMP/ATT）和信道
- 顶部控制栏：[▶ 自动播放] [⏸ 暂停] 速度选择(0.5x/1x/2x)
- 自动播放：每 2~3 秒高亮下一步，自动滚动

**步骤展开内容**：逐字段完整解析（字段名、大小、值示例、含义说明），包含数据经过的协议层标注。

### 组件三：五合一速查表

**目标**：手册第十章的五张速查表，支持快速检索。

**实现**：Tab 切换 + 即时搜索过滤。

**Tab**：LL Control Opcode | SMP Code | ATT Opcode | L2CAP CID | AD Type

**搜索**：
- 输入即时过滤当前 Tab 表格行
- 匹配文字高亮黄色标记
- 空输入显示全部

### 组件四：流程图与协议栈

**协议栈图（第一章）**：用 CSS flexbox 绘制 7 层堆叠块，每层标注协议名 + 一句话。点击跳到对应章节。

**配对四阶段流程图（第七章）**：纯 CSS 节点 + 连线，Phase 1→2→3→4 纵向排列，节点带 ▸ 可点击展开内部详情。

**四种认证路径分支图（第七章）**：CSS flexbox 三列分支，箭头连接。

**加密范围标注图（第九章）**：在色块条表示空中帧的基础上，用文字标注每个字段"明文"还是"密文"，并注明原因。

## 数据架构

### 内容分工：静态 HTML vs JS 动态渲染

页面内容分两类，按性质选择渲染方式：

| 内容类型 | 渲染方式 | 示例 |
|---------|---------|------|
| 说明性文案（段落、注释、列表） | 静态 HTML，直接写在 `<section>` 中 | 章节引言、概念解释、"阅读建议" |
| 结构化数据（表格、位布局、时序步骤） | JS 数据对象 + 渲染函数生成 DOM | 位布局色块条、Opcode 表、27 步时序 |
| 混合型（流程图 + 表格组合的章节） | HTML 骨架中放占位元素（`<div data-component="...">`），JS 扫描替换 | 配对四阶段、加密三次握手、GATT 层级图 |

占位标记约定：`<div data-component="bit-layout" data-id="adv-pdu-header"></div>`，页面加载后 `renderBitLayout()` 扫描所有 `[data-component="bit-layout"]` 并替换为完整组件。

### 单文件结构

```
<!DOCTYPE html>
<head><style>/* ~800 行 CSS */</style></head>
<body>
  <!-- ~300 行 DOM：顶栏 + 侧栏 + 各章节 <section>（含文案 + 占位标记） -->
  <script>
  // 层1: 数据定义 (~1200 行，只存结构化数据，文案已在 HTML 中)
  const BIT_LAYOUTS = { ... }    // 位布局定义
  const TIMING_STEPS = [ ... ]   // 27 步序列
  const TREE = [ ... ]           // 导航树（含 anchors 映射）
  const REF_TABLES = { ... }     // 速查表数据

  // 层2: 渲染函数 (~400 行)
  function renderBitLayout() { ... }
  function renderTimeline() { ... }
  function renderRefTable() { ... }

  // 层3: 交互控制器 (~200 行)
  // 导航、主题、搜索、展开折叠
  </script>
</body>
```

### 关键数据结构

**位布局**（`BIT_LAYOUTS`）：
```js
{
  id: "adv-pdu-header",
  title: "广告信道 PDU Header（16 bit）",
  fields: [
    { name, bits, range: [start, end], color, desc, values: [...] },
    // ...
  ]
}
```

**时序步骤**（`TIMING_STEPS`）：
```js
{
  id: 1, from: "B", to: "A",
  phase: "广播阶段",
  channel: "37/38/39",
  title: "ADV_IND",
  summary: "PDU Type=0000, AdvA=MAC_B",
  layer: "LL",
  detailLayout: "adv-ind-payload"  // 引用 BIT_LAYOUTS key
}
```

### 渲染策略

- 页面加载：一次性渲染全部内容到 DOM
- 导航点击：滚动定位到对应 `<section>`，非切换显示
- 搜索过滤：直接操作 DOM 行 `display`
- 不使用虚拟 DOM

## 关键数据准确性约束（来自 HANDOFF.md）

- PDU Type 是 4 bit，不是 2 bit
- 广播 PDU Header Length 字段用 8 bit（BLE 4.2+）
- LL_START_ENC_REQ/RSP 本身是明文
- SMP Confirm (0x03) 在 SC 和 Legacy 中含义不同
- CONNECT_IND 后从设备不回复 ACK
- Access Address 不是 PDU 字段，是帧头
- 加密协商全过程不走广播包
- CCCD 连接断开自动归零
