# 6.3 GATT 数据组织

## 一、ATT vs GATT

ATT = 动词（怎么读写），GATT = Schema（数据怎么组织）

## 二、四级树形结构

```
Profile（概念，不存在于 Attribute DB）
└── Service（功能模块集合）
    └── Characteristic（具体数据项）
        ├── Declaration [Properties][Value Handle][UUID]
        ├── Value（实际数据）
        └── Descriptor（附加配置）
```

## 三、各层详解

### Service Declaration
Type=0x2800(Primary) 或 0x2801(Secondary)，Value=Service UUID

### Characteristic Declaration
Type=0x2803，Value=[Properties(1B)][Value Handle(2B)][UUID]

Properties 位掩码：Broadcast/Read/WriteWithoutResponse/Write/Notify/Indicate/SignedWrite/ExtProps

### CCCD（最重要的 Descriptor）
Type=0x2902，Value=0x0000(关)/0x0001(Notify)/0x0002(Indicate)

必须写 CCCD 才开启 Notify/Indicate。连接断开自动重置为 0x0000。

### 其他 Descriptor
- CUD(0x2901)：用户可读名称
- Presentation Format(0x2904)：数据格式（单位/指数）
- Valid Range(0x2906)：有效范围

## 四、流量计 GATT 数据库实例

```
Battery Service (0x180F)
  └─ Battery Level [Read|Notify], Value=85%
      └─ CCCD

Device Info Service (0x180A)
  └─ Manufacturer Name [Read], Value="ACME Co."

流量计自定义 Service (128-bit UUID)
  ├─ 流速 [Read|Notify], CCCD已开启
  ├─ 采样间隔 [Read|Write], Value=5s
  ├─ 累计流量 [Read|Notify], CCCD已开启
  └─ 校准参数 [Read|Write], Value=(100字节)
```

## 五、对开发的影响

- Properties 决定操作能力 → `getProperties()`
- 订阅通知必须先写 CCCD
- Value Handle 是 Declaration 属性之一，串联 Declaration 和 Value
- 好的固件提供 CUD/Presentation Format 改善 UI 显示
