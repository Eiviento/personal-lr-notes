# 6.4 UUID 体系

## 一、什么是 UUID

128-bit 全球唯一标识符。BLE 用 UUID 标识 Service/Characteristic/Descriptor。

## 二、16-bit vs 128-bit UUID

蓝牙 SIG 预分配 16-bit UUID，放入固定的 128-bit Base UUID：

```
Base UUID: 0000XXXX-0000-1000-8000-00805F9B34FB
           ↑ 16-bit UUID 填入这里

例：0x180F → 0000180F-0000-1000-8000-00805F9B34FB
```

优点：协议包更小、广播数据省空间、全局不冲突。

## 三、U UID 分配规律

| 前缀 | 类别 | 示例 |
|------|------|------|
| 0x18XX | Service | 0x180F=Battery, 0x181A=Env Sensing |
| 0x2AXX | Characteristic | 0x2A19=Battery Level, 0x2A6E=Temperature |
| 0x29XX | Descriptor | 0x2902=CCCD, 0x2901=CUD |

## 四、仪器仪表相关标准 UUID

0x181A (Env Sensing) / 0x2B18 (Volume Flow Rate, 5.3) / 0x2A58 (Analog Input) / 0x2A5C (Digital Output)

## 五、厂商自定义 128-bit UUID

用 UUID v4 随机生成（uuidgen / [guid]::NewGuid()），不要手动编。
建议：公司统一 Base UUID，各 Service/Characteristic 从 Base UUID 派生。

## 六、混搭策略

- Device Info / Battery → 16-bit 标准（任何工具可解析）
- 流量计核心 Service + 特征 → 128-bit 自定义
- CCCD → 16-bit 标准（0x2902，规范强制）
- Android: `UUID.fromString("0000180f-0000-1000-8000-00805f9b34fb")`
