# K05 — 编码层 vs 封装层

> 学习日期：2026-07-11
> 前置依赖：K01

---

## 1. 形象类比

```
书的内容文字 = 编码层（Codec） → 视频数据本身如何被压缩
书的封面装订 = 封装层（Container） → 压缩后的数据如何打包成文件

同一本书可以用精装、平装、电子书出版——
同样的 H.264 编码流可以用 MP4、MKV、TS 封装。
```

---

## 2. 编码层（Codec = Coder + Decoder）

编码层决定了视频如何被压缩、压缩率多大、解码需要多少算力。

演进树：MPEG-1 → MPEG-2 → MPEG-4 Part 2 → **H.264/AVC**（2003，至今主宰） → H.265/HEVC（2013，4K） → H.266/VVC（2020） + **AV1**（2018，开源）

---

## 3. 封装层（Container）

容器是"箱子"，里面可以装多个流：

```
video.mp4
├─ 视频流（H.264, 1920×1080, 30fps）
├─ 音频流 #1（AAC, 128kbps）
├─ 音频流 #2（AC3, 5.1 环绕声）
├─ 字幕流（中/英）
└─ 元数据（创建时间、GPS、旋转角度...）
```

| 容器 | 扩展名 | 特点 |
|------|--------|------|
| MP4 | .mp4, .m4v | 兼容性之王 |
| MKV | .mkv | 几乎能装任何编码，开源 |
| AVI | .avi | 古老，不推荐 |
| MOV | .mov | Apple 生态 |
| TS | .ts | 广播和 HLS 流媒体 |
| FLV | .flv | RTMP 直播 |
| WebM | .webm | 专为 Web 设计 |

---

## 4. 核心认知

```
❌ ".mp4 = H.264" — 错误
✅ "这个 mp4 文件里装的是 H.264 视频流" — 正确

.mp4 里视频可以是 H.264、H.265、MPEG-4 等
编码 ≠ 封装，两个独立维度
```

---

## 5. 为什么这个区分重要

直接引出后续核心概念：

- **转封装（K13）**：只换容器，编码数据原封不动 → 快速无损
- **转码（K15）**：解码→重编码→封装 → 慢速有损

---

## 6. 查看工具

- `ffprobe -v quiet -print_format json -show_format -show_streams video.mp4`
- MediaInfo GUI：Format（封装层） + Codec ID（编码层）
