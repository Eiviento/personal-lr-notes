## 1. H.264 是谁

编码标准谱系：MPEG-1(1993) → MPEG-2(1995) → MPEG-4 Part 2(1999) → **H.264/AVC(2003)** → H.265/HEVC(2013) → H.266/VVC(2020) / AV1(2018)

- H.264 = AVC = MPEG-4 Part 10，同一个标准三个名字
- ITU-T 制定（H.264） + ISO/IEC 制定（MPEG-4 Part 10/AVC）
- 至今地球上使用最广泛的视频编码

---

## 2. 核心创新：利用时间冗余

```
MJPEG 思路：每帧独立压缩，100 帧 × 100KB = 10 MB
H.264 思路：第一帧完整编码（100KB），后续帧只记录差异（1KB）
         总计：100KB + 99×1KB ≈ 0.2 MB → 50 倍压缩
```

在 K02-K03 的空间压缩基础上，新增**帧间预测**——大多数相邻帧差异极小，只记录"变化的部分"。

---

## 3. 编码流程（五步）

```
输入 YUV 帧
  → ① 预测（帧内预测 + 帧间预测） ← 贡献大部分压缩率
  → ② 残差 = 预测值 - 真实值
  → ③ 变换（DCT，空间域→频域）
  → ④ 量化（丢高频信息） ← "有损"的主要来源
  → ⑤ 熵编码（CABAC/CAVLC） ← 数学无损的最后压缩
  → 输出 H.264 比特流
```

解码器反向：熵解码 → 反量化 → 反变换 → 加预测值 → 重建帧

---

## 4. 宏块与预测

一帧 = 120×68 个 16×16 宏块（1920×1080），编码器逐块找最省钱的编码方式：

- **帧内预测**：参考同帧相邻块，"我跟左边一样" + 微小差值
- **帧间预测**：参考前后帧的块，"我从(x,y)移到了(x+Δx,y+Δy)"（运动向量）

---

## 5. 后四步详解

| 步骤 | 做什么 | 要点 |
|------|--------|------|
| 残差 | 预测值 ≠ 真实值的差值 | 残差小而接近零 = 高压缩率来源 |
| 变换 | 整数 DCT，空间域→频域 | 分离低频（重要）和高频（可丢弃） |
| 量化 | 对高频粗量化 | QP 控制质量，QP 越大压缩越高画质越低 |
| 熵编码 | 数学压缩 | CAVLC（简单快速）/ CABAC（压缩率高 10-15%） |

---

## 6. NAL Units 数据组织

H.264 输出 = 一串 NALU：

```
每个 NALU = [NAL 头 1-2B] + [载荷 RBSP]

关键 NALU 类型：
  类型 1：非 IDR slice（普通 P/B 帧数据）
  类型 5：IDR slice（I 帧数据）
  类型 7：SPS（序列参数集——分辨率、Profile、Level）
  类型 8：PPS（图像参数集——熵编码模式、量化初始值）

SPS 和 PPS 必须在数据流最前面，解码器没有它们无法初始化。
```

---

## 核心公式

> 压缩 = 预测（找相似） + 残差（只存差异） + 量化（丢不重要信息） + 熵编码（数学压缩）
