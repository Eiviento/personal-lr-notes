## 1. 帧内预测（Intra Prediction）

不直接存储像素，而是说"和左边/上边一样，按这个规律延伸"。

- 用已编码的上方和左方像素预测当前 4×4/16×16 块
- 4×4 亮度块有 **9 种预测模式**（垂直、水平、DC 均值、各角度对角线...）
- 编码器尝试全部模式，选残差最小的
- 存储：`[模式编号 3bit] + [残差]`

---

## 2. 帧间预测（Inter Prediction）

### 运动估计
在参考帧搜索窗口内逐像素比对，用 SAD（绝对差值和）找最相似的块：

```
SAD = Σ|当前块像素(i,j) - 参考块像素(i,j)|
SAD 越小 = 越相似 = 残差越小
```

### 运动补偿
输出：**运动向量**（"块从 (100,50) 移到了 (95,48)"） + **残差**（新位置的像素微调）

### 计算代价
1080p ≈ 8160 宏块 × 每个比对几千候选位置 × 256 像素 = 每秒 1200 亿次操作。这就是为什么视频编码是计算密集型任务。

---

## 3. 变换（整数 DCT）

残差块 4×4，从空间域→频域。能量集中到左上角（低频），右下角（高频）大多归零。

---

## 4. 量化 ← 唯一的有损步骤

```
DCT 系数 ÷ 量化步长（由 QP 控制）
QP 小 → 保留细节多 → 画质高
QP 大 → 大量归零 → 压缩比高、画质低
```

不可逆，被量化为 0 的信息永久丢失。

---

## 5. 熵编码

量化后矩阵 → zigzag 扫描 → 一维序列（非零集中在前，后面一长串 0）

| | CAVLC | CABAC |
|--|-------|-------|
| 方式 | 查表映射 | 算术编码（概率计算） |
| 压缩率 | 基准 | 高 10-15% |
| 速度 | 快 | 慢 |
| Profile | Baseline | Main 及以上 |
| 优势 | 分配整数 bit | 可分配非整数 bit（0.6/1.3 bit） |

---

## 6. 去块效应滤波（Deblocking Filter）

- 消除宏块边界"接缝"（不同宏块量化参数不同导致）
- **编码循环内**进行（不是后处理）
- 滤波后的帧用作后续帧的参考——避免块效应恶性循环

---

## 完整流程

```
预测（找相似）→ 残差（只存差异）→ DCT（频域分离）
→ 量化（丢高频，唯一有损）→ 熵编码（数学压缩）→ 去块滤波（平滑+改进参考）
```
